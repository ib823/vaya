ðŸ“‹ Common Incident Playbooks

PLAYBOOK: API Latency Spike

  1. Check VayaMetrics dashboard for affected endpoints
  2. Review recent deployments (rollback if correlation)
  3. Check database query performance
  4. Check cache hit rates
  5. Check supplier API response times
  6. Scale horizontally if load-related

PLAYBOOK: Payment Failures

  1. Check payment processor status pages
  2. Review error codes in logs
  3. Check if specific payment method affected
  4. Contact payment processor if their issue
  5. Enable backup payment processor if available
  6. Notify affected users with ETA

PLAYBOOK: Database Issues

  1. Check VayaDB health metrics
  2. Check disk space and I/O
  3. Review slow queries
  4. Check replication lag
  5. Consider failover to replica if primary issues
  6. Enable read-only mode if write issues

ðŸ“‹ Common Issue Scripts

1. Apologize for the inconvenience 2. Verify booking reference and email 3. Check booking status in system 4. If payment successful but no confirmation: - Check supplier booking status - If confirmed: resend confirmation email - If pending: escalate to supplier - If failed: initiate refund + offer rebooking 5. Document interaction and resolution

SCRIPT: Refund Request

1. Verify identity and booking details 2. Check fare rules and refund eligibility 3. Explain refund amount calculation 4. If eligible: initiate refund, provide timeline 5. If not eligible: explain why, offer alternatives - VAYA Credit with bonus - Date change (if cheaper than cancel) 6. Confirm customer understanding 7. Document interaction

SCRIPT: Oracle Prediction Wrong

1. Acknowledge their frustration 2. Explain predictions are estimates, not guarantees 3. Review their specific case: - What was predicted vs actual? - Were there market events? 4. Offer goodwill gesture if appropriate: - Free month of Plus - VAYA Credit 5. Log feedback for ML team 6. Thank them for helping improve our predictions
